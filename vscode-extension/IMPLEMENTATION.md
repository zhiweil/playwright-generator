## Architecture

The extension uses the VS Code **WebviewView** API to render a persistent panel in the Activity Bar sidebar. All UI runs inside a sandboxed webview (HTML/CSS/JS), communicating with the extension host via `postMessage`.

## Key Files

### `src/extension.ts`
Entry point. Registers `PlaywrightGeneratorPanel` as a `WebviewViewProvider` for the view ID `playwrightGenerator.panel`, and registers the `playwrightGenerator.open` command. `retainContextWhenHidden: true` keeps the webview state when the panel is collapsed.

### `src/panel.ts`
- `resolveWebviewView` вҖ” sets up the webview HTML, message listener, file watchers, and sends initial data
- `_sendInitialData` вҖ” reads `.env`, scans TC IDs and tags, posts `init` message to the webview
- `_handleMessage` вҖ” handles `saveEnv`, `generate`, `runAll`, `runByTag`, `runHeaded`, `debug`, `report` messages by writing `.env` or spawning a VS Code terminal
- `_setupWatchers` вҖ” creates two `FileSystemWatcher` instances; on any change in `tests/` or `generated/` it rescans and posts `updateTestCaseIds` or `updateTags` to the webview
- `_runInTerminal` вҖ” creates a named terminal, `cd`s to the workspace root, and sends the command

### `src/envManager.ts`
- `readEnv(workspaceRoot)` вҖ” parses the `.env` file line by line, strips inline comments, returns an `EnvConfig` object with defaults for any missing keys
- `writeEnv(workspaceRoot, config)` вҖ” updates existing key=value lines in place, preserving comments and unknown keys; appends any keys not already present

### `src/testCaseScanner.ts`
- `scanTestCaseIds(workspaceRoot)` вҖ” walks `tests/**/*.test.md`, extracts all `[TC-*]` tags using regex, returns sorted deduplicated list
- `scanTags(workspaceRoot)` вҖ” walks `generated/**/*.test.ts`, extracts all `[TAG]` bracket tags from test titles, returns sorted deduplicated set

### `media/main.js`
Vanilla JS running inside the webview sandbox. Responsibilities:
- Handles `init`, `updateTestCaseIds`, `updateTags` messages from the extension host
- Shows/hides model-specific fields when the AI Model dropdown changes
- Filters TC ID and tag lists as the user types in the search boxes
- Syncs the search box value when the user clicks a list item
- Posts messages back to the extension host on button clicks

### `media/style.css`
Uses VS Code CSS custom properties (`--vscode-*`) so the panel automatically matches the user's colour theme (light, dark, high contrast).

## Message Protocol

| Direction | Command | Payload |
|---|---|---|
| Extension вҶ’ Webview | `init` | `{ env, testCaseIds, tags }` |
| Extension вҶ’ Webview | `updateTestCaseIds` | `{ testCaseIds }` |
| Extension вҶ’ Webview | `updateTags` | `{ tags }` |
| Webview вҶ’ Extension | `saveEnv` | `{ env: EnvConfig }` |
| Webview вҶ’ Extension | `generate` | `{ tcId }` |
| Webview вҶ’ Extension | `runAll` | вҖ” |
| Webview вҶ’ Extension | `runByTag` | `{ tag }` |
| Webview вҶ’ Extension | `runHeaded` | `{ tag }` |
| Webview вҶ’ Extension | `debug` | `{ tag }` |
| Webview вҶ’ Extension | `report` | вҖ” |

## Development

### Prerequisites

```bash
npm install -g @vscode/vsce
```

### Run in Debug Mode

1. Open the `playwright-generator/` folder in VS Code
2. Press `F5` вҖ” this runs the `build-extension` task then launches the Extension Development Host
3. The Playwright Generator icon appears in the Activity Bar of the new window
4. Open a workspace that contains a `.env` file to activate the extension

### Build

```bash
cd vscode-extension
npm install
npm run build
```

### Watch Mode

```bash
npm run watch
```

## Publishing to the VS Code Marketplace

### 1. Create a Publisher Account

1. Go to https://marketplace.visualstudio.com/manage
2. Sign in with a Microsoft account
3. Create a publisher with ID `zhiweiliu`

### 2. Create a Personal Access Token (PAT)

1. Go to https://dev.azure.com
2. Profile icon вҶ’ **Personal access tokens** вҶ’ **New Token**
3. Set **Organization** to **All accessible organizations**
4. Set **Scopes** to **Marketplace вҶ’ Manage**
5. Copy the token

### 3. Login

```bash
cd vscode-extension
vsce login zhiweiliu
# paste PAT when prompted
```

### 4. Add the Marketplace Icon

The Activity Bar uses `media/icon.svg`. The Marketplace listing requires a separate 128Г—128 PNG at `media/icon.png`. Add this file before publishing.

### 5. Publish

```bash
# Build first
npm run build

# Preview what will be packaged
vsce ls

# Package to a .vsix file (optional вҖ” inspect before publishing)
vsce package

# Publish to the Marketplace
vsce publish
```

### 6. Publish a New Version

```bash
vsce publish patch   # 0.0.1 вҶ’ 0.0.2
vsce publish minor   # 0.0.1 вҶ’ 0.1.0
vsce publish major   # 0.0.1 вҶ’ 1.0.0
```

### 7. Unpublish

```bash
vsce unpublish zhiweiliu.playwright-generator-vscode
```

## Notes

- The `dist/` folder is generated by `tsc` and should be added to `.gitignore`
- The `node_modules/` folder should also be in `.gitignore`
- The `punycode` deprecation warning during build comes from the TypeScript compiler itself and is harmless вҖ” it does not affect the extension at runtime
- The Activity Bar icon must be an SVG with `fill="currentColor"` вҖ” PNG is not supported for the Activity Bar
